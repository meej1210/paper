# 前端美化计划（归档）

> 说明：此文件为基于本次会话已读取内容的恢复版，保留原计划结构，供后续 UI 迭代参考。

## 背景

当前前端已经具备基础功能，但整体仍偏向 MVP 原型状态，主要问题集中在以下几个方面：

- 页面逻辑割裂，用户主流程不够顺畅，存在“会用但不好用”的情况。
- 页面视觉较朴素，缺少统一的产品化 UI / UX 语言。
- 动效反馈不足，交互显得生硬，界面缺少节奏感与灵动感。
- 缺少合适的静态视觉素材支撑，首页、创建页、登录页的氛围感和完成度不足。

本计划将前端优化拆分为四轮迭代，按“先理顺逻辑，再升级视觉，再补动效，最后落静态图”的顺序推进，减少返工，并让每一轮都能独立交付可见成果。

---

## 总目标

- 让用户第一次进入系统时，能快速理解产品用途与核心操作路径。
- 让主要任务链路更顺滑：登录 -> 进入工作台 -> 创建扫描 -> 查看执行状态 -> 查看结果 -> 导出报告。
- 让页面从“能用”升级到“像正式产品”。
- 让交互更自然、有反馈、有节奏。
- 为后续接入静态图和答辩展示留出统一的视觉容器。

## 当前问题总结

### 1. 页面逻辑问题

- 顶部导航更像路由链接集合，不像产品导航。
- 任务列表页是“裸表格”，缺少工作台感和主操作引导。
- SAST / DAST 创建页像后台管理表单，缺少引导和反馈。
- 详情页信息密度过高，首屏不能快速给出结论。
- 过滤条件存在“边输入边请求”的行为，容易让用户觉得页面在乱跳。
- 任务创建后直接跳转详情页，缺少清晰的成功反馈和下一步预期。

### 2. 视觉问题

- 页面风格偏基础，缺少统一视觉体系。
- 页面层级感不足，卡片、按钮、标题、数据之间的主次不够清晰。
- 缺少系统级品牌感和产品气质。

### 3. 动效问题

- 现有界面反馈偏弱，缺少状态切换时的节奏感。
- 页面进入、表单提交、任务状态变化等场景可以补充更有目的性的动效。
