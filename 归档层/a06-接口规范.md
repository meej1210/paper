## 1. 规范目标

- 固定前后端联调接口结构
- 优先保证 MVP 主线接口可直接开发
- 降低后续返工成本

接口设计原则：

- 使用 RESTful 风格
- 所有接口统一使用 `/api` 前缀
- 除注册、登录、健康检查外，其余接口默认需要 Token
- 响应结构统一
- 错误码统一

---

## 2. 基础约定

### 2.1 Base URL

本地开发建议：

```text
http://localhost:5000/api
```

### 2.2 鉴权方式

请求头：

```http
Authorization: Bearer <access_token>
```
