## 2. 推荐运行方式

### 2.1 一键启动

```powershell
cd D:\ziliao\dev
powershell -ExecutionPolicy Bypass -File .\start_demo_env.ps1
```

脚本会自动完成以下动作：

- 检查 `D:\anaconda\python.exe`、Docker、Node.js 与前端 Vite 入口是否存在
- 创建运行状态目录：`tmp\run-control`
- 创建日志目录：`tmp\run-control\logs`
- 启动或复用 Redis
- 优先在 `3000` 端口启动 Juice Shop；若 `3000` 被占用且不是 Juice Shop，则回退到 `3100`
- 启动后端 API、Celery Worker、前端 Vite，并等待健康检查通过

启动成功后，重点查看：

- 状态文件：`tmp\run-control\demo-runtime.json`
- API 日志：`tmp\run-control\logs\api.out.log`、`tmp\run-control\logs\api.err.log`
- Worker 日志：`tmp\run-control\logs\worker.out.log`、`tmp\run-control\logs\worker.err.log`
- 前端日志：`tmp\run-control\logs\frontend.out.log`、`tmp\run-control\logs\frontend.err.log`

### 2.2 一键停止

```powershell

```

停止脚本会：

- 按 `demo-runtime.json` 中记录的 PID 关闭已托管的 API / Worker / Frontend 进程
- 删除运行状态文件
- 执行 `docker compose down`
- 如果曾使用 `3100` 端口回退容器，还会删除 `devsecops-juice-shop-3100`

## 4. 常见排障方向

### 4.1 PDF 导出失败
- 优先检查 WeasyPrint 及 GTK / Pango 原生依赖
- 在 Windows 环境下，这通常不是业务代码问题，而是运行时依赖问题
- 可先执行 `backend\check_runtime_deps.py`
- 若仍失败，优先查看 `tmp\run-control\logs\api.err.log`

### 4.2 DAST 无法执行
- 检查 Juice Shop 是否在线
- 检查目标是否命中白名单
- 检查 Wapiti 命令是否可用
- 检查后端健康接口中的 `dast_scanner`
- 若 `3000` 被占用，确认脚本是否回退到了 `3100`

### 4.3 异步任务不消费
- 检查 Redis 是否启动
- 检查 Worker 是否在线
- 检查 `CELERY_TASK_ALWAYS_EAGER` 是否误设为同步模式
- 查看 `worker.out.log` 是否列出了：
  - `app.workers.dast_tasks.execute_dast_task`
  - `app.workers.sast_tasks.execute_sast_task`
- 查看 `worker.err.log` 中是否出现 `Connected to redis://localhost:6379/0` 与 `ready`

### 4.4 前端构建 / 页面行为异常
- 先跑 `npm run build`
- 再重点检查 `TaskDetailView.vue`、路由跳转与全局错误提示
- 若是联调页面不可访问，查看 `frontend.out.log` 是否出现 `http://127.0.0.1:5173/`

### 4.5 启动脚本失败

- 若提示已有托管进程存在，先执行 `.\\stop_demo_env.ps1`
- 若提示 `5000`、`5173`、`3000`、`3100` 端口冲突，先确认是否已有可复用服务
- 若提示 Vite 入口不存在，先在 `frontend` 目录执行 `npm install`
