# 失败场景说明

> 说明：原文件在损坏前已出现大量乱码；此文件依据本次会话中可辨识结构与 TODO 引用重建为可用版。

本文档用于收口当前已覆盖与待补的失败场景，重点覆盖非法输入、非白名单目标、超时、扫描器失败、导出依赖缺失等问题。

## 一、SAST 失败场景

### 1. 非法文件类型
- 接口：`POST /api/sast/tasks`
- 输入：例如 `malware.exe`
- 预期：返回 `400`
- 当前依据：`backend/tests/test_tasks.py`

### 2. 空文件 / 非 Python 内容
- 目标：验证上传校验与扫描器前置处理是否合理
- 建议覆盖：
  - 空 Python 文件
  - 空 zip
  - 解压后无可扫描目标

### 3. 压缩包目录穿越
- 目标：验证安全解压逻辑是否拦截 `../` 等路径穿越输入
- 当前状态：属于应重点保留的安全边界场景

### 4. 扫描器执行失败
- 目标：验证 Bandit 缺失、执行异常或 PATH 配置错误时的失败表现
- 关注点：任务状态、错误摘要、失败日志、前端提示

## 二、DAST 失败场景

### 1. 非法 URL 协议
- 接口：`POST /api/dast/tasks`
- 输入：例如 `ftp://bad.scheme.com`
- 预期：返回 `400`
- 当前依据：`backend/tests/test_tasks.py`

### 2. 非白名单目标
- 接口：`POST /api/dast/tasks`
- 输入：例如 `http://example.com`
- 预期：返回 `400`
- 当前依据：`backend/tests/test_tasks.py`

### 2.1 显式公网白名单目标
- 接口：`POST /api/dast/tasks`
- 配置：`DAST_ALLOWED_HOSTS` 包含目标主机，例如 `authorized.example.test`
- 输入：例如 `https://authorized.example.test`
- 预期：允许创建任务，`target_policy` 为 `allowed_host`
- 当前依据：`backend/tests/test_tasks.py::TestDastTaskCreation::test_create_dast_allows_explicit_public_host`
- 说明：这是后续支持公网授权 URL 的最小闭环，不等于允许任意公网扫描。

### 2.2 未勾选授权确认
- 接口：`POST /api/dast/tasks`
- 输入：合法本地 URL，但缺少 `authorization_confirmed=true`
- 预期：返回 `400`，错误字段为 `authorization_confirmed`
- 当前依据：`backend/tests/test_tasks.py::TestDastTaskCreation::test_create_dast_requires_authorization_confirmation`

### 3. 超时参数非法
- 接口：`POST /api/dast/tasks`
- 输入：例如 `timeout=99999`
- 预期：返回 `400`
- 当前依据：`backend/tests/test_tasks.py`

### 4. 扫描超时
- 目标：验证 Wapiti 任务超时后是否正确进入 `TIMEOUT`
- 关注点：错误信息、任务摘要、前端可见反馈

### 5. 扫描器失败
- 目标：验证 Wapiti 缺失或执行异常时任务是否进入 `FAILED`
- 关注点：后端错误记录 + 前端失败提示完整性

## 三、导出与依赖失败场景

### 1. PDF 依赖缺失
- 典型问题：Windows 下 WeasyPrint 缺少 GTK / Pango 原生依赖
- 当前结论：此类问题应在环境检查阶段尽早暴露，而不是等到现场导出时报错

### 2. 报告文件缺失
- 目标：验证任务已完成但报告原文件丢失时的接口与前端表现

## 四、当前建议

- 对已被 `backend/tests` 覆盖的输入校验场景，保留自动化测试即可
- 对真实扫描器、超时、导出依赖类问题，必须保留真实联调记录
- 对前端失败提示，应补齐 401 / 403 / 500、导出失败、AI 解读失败、任务取消失败等反馈
