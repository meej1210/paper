### DAST 多靶场采样计划

当前 DAST 样本已从单一 Juice Shop 扩展为本地多靶场矩阵，建议后续采样时按固定参数执行：

| 靶场 | URL | 建议轮次 | 推荐超时 | 用途 |
|---|---|---:|---:|---|
| Juice Shop | `http://127.0.0.1:3000` | 3 | 180 秒 | 现代 Web / SPA 基线 |
| DVWA | `http://127.0.0.1:3001` | 3 | 180 秒 | 传统漏洞样本 |
| WebGoat | `http://127.0.0.1:3002/WebGoat` | 3 | 180 秒 | 训练型漏洞样本 |
| Mutillidae II | `http://127.0.0.1:3003` | 3 | 180 秒 | 综合传统 Web 样本 |

采样命令：

```powershell
cd D:\ziliao\dev
$env:DEVSECOPS_USER="admin@qq.com"
$env:DEVSECOPS_PASSWORD="admin"
D:\anaconda\python.exe scripts\run_dast_lab_samples.py `
  --base-url http://127.0.0.1:5000/api `
  --username $env:DEVSECOPS_USER `
  --password $env:DEVSECOPS_PASSWORD `
  --targets-file 主文档层\dast_targets.local.json `
  --rounds 3 `
  --scan-timeout 180 `
  --out-dir tmp\dast-lab-samples
```

采样完成后，将 `tmp\dast-lab-samples\summary.md` 中的结果补充到本文件的正式统计表中。

### DAST 公网授权样本计划

公网 URL 只作为“显式授权样本”采集，不纳入默认样本池。推荐记录字段：

| 字段 | 示例 |
|---|---|
| 目标 URL | `https://security-test.example.com` |
| 白名单配置 | `DAST_ALLOWED_HOSTS=127.0.0.1,localhost,security-test.example.com` |
| 授权说明 | 本人或团队拥有的测试站点 |
| 命中策略 | `allowed_host` |
| 授权确认 | `authorization_confirmed=true` |

默认拒绝样例仍保留 `https://example.com`，用于证明系统不会扫描未授权公网目标。
