# SCA 演示预期发现

本文档用于演示时辅助讲解。不同 SCA 工具和漏洞数据库的命中结果不完全相同，以下内容应作为“预期风险点”和“讲解脚本”，最终以平台实际扫描报告为准。

## 一、演示目标

通过扫描 `tests/sca_demo`，展示平台能够完成以下工作：

1. 自动识别 pip、npm、Maven 三类依赖清单。
2. 提取组件名称、版本号、生态类型和依赖来源文件。
3. 关联漏洞编号，例如 CVE、GHSA、PYSEC、OSV。
4. 按严重级别聚合高危、中危、低危风险。
5. 输出修复建议，例如升级到不受影响的版本。
6. 支持报告导出或整改复扫闭环。

## 五、建议展示的系统能力

### 1. 资产识别

先展示平台能从多个清单中识别组件，并保留来源文件路径。例如同一个报告中同时出现：

- Python: `requirements-vulnerable.txt`
- Node.js: `package-vulnerable.json`
- Java: `pom-vulnerable.xml`

### 2. 风险分级

按严重级别筛选报告，优先展示高危漏洞，例如 Log4j、Jackson、PyYAML、lodash 等组件。说明高危风险应进入优先整改队列。

### 3. 修复建议

打开单条漏洞详情，展示平台给出的安全版本或升级范围。讲解整改过程：

1. 升级依赖版本。
2. 运行单元测试和兼容性测试。
3. 重新执行 SCA 扫描。
4. 确认漏洞状态从未修复变为已修复或不再命中。

### 4. 误报和例外管理

如果平台支持忽略或例外审批，可选择一个低危或开发依赖风险，演示备注原因、设置过期时间、保留审计记录。

### 5. 报告导出

导出 PDF、Excel 或 JSON 报告，用于展示：

- 漏洞编号
- 影响组件
- 当前版本
- 修复版本
- 严重级别
- 漏洞描述
- 整改负责人或状态

## 六、验收标准

演示系统效果时，建议至少确认以下结果：

- 能识别不少于 3 种依赖清单格式。
- 能展示不少于 15 个第三方组件。
- 能命中多个严重级别的漏洞。
- 能在漏洞详情中看到组件名、当前版本和建议修复版本。
- 能按生态、严重级别或来源文件筛选。
- 能导出或保存扫描报告。

## 七、演示口径

推荐一句话总结：

> 这个测试用例模拟了一个长期未治理的多语言项目，SCA 扫描可以快速识别开源组件资产、定位已知漏洞并给出升级建议，从而把供应链风险纳入 DevSecOps 闭环管理。
