# SCA 演示测试用例设计

## 背景

当前 DevSecOps 平台已支持 SCA 依赖漏洞扫描。后端接收 `requirements.txt` 或包含 `requirements.txt` 的 ZIP，调用 `pip-audit` 生成 JSON 报告，并在任务详情中展示依赖数量、漏洞数量、可修复数量、包名、当前版本、漏洞 ID 和修复版本。

## 目标

新增一个可复现的 SCA 演示输入文件，并补充中文测试用例与演示说明，便于答辩、论文截图、平台功能展示和 CI/CD 门禁演示。

## 范围

- 新增 `tests/sca_demo/requirements-vulnerable.txt`，作为含旧版本依赖的 SCA 演示样例。
- 新增 `tests/sca_demo/README.md`，说明样例用途和安全边界。
- 新增 `主文档层/a26-SCA测试用例与演示说明.md`，覆盖测试目标、前置条件、Web 演示步骤、CI/CD 命令、预期结果、展示话术和截图建议。

## 非目标

- 不修改 SCA 后端扫描逻辑。
- 不新增 NPM、Maven、Go 等其他生态的 SCA 示例。
- 不自动安装演示依赖。
- 不保证每个漏洞 ID 永久不变，因为 `pip-audit` 使用的漏洞库会随时间更新。

## 验证方式

- 确认新增样例文件路径存在，且可被平台作为 SCA 上传目标。
- 通过平台上传样例文件，任务应进入成功或可解释的失败状态。
- 若扫描成功，详情页应展示依赖漏洞和修复版本。
- 通过 `scripts/ci_security_scan.ps1` 运行 SCA 门禁时，存在可修复漏洞应返回退出码 `3`。

## 约束

当前目录不是 git 仓库，因此本设计文档无法提交到 git。实施时只创建本地文件并进行可执行性检查。
